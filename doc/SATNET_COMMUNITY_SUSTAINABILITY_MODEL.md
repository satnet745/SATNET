# SATNET Community Sustainability Model

## Purpose

This document aligns SATNET's sustainability strategy with open-source community norms and cypherpunk principles.

The goal is not to maximize extraction from users. The goal is to keep the network alive, forkable, auditable, and financially sustainable without compromising self-custody or user freedom.

---

## Core Principles

1. **No protocol tax**
   - SATNET should not impose a mandatory protocol-level fee on every voucher, payment, or wallet action.

2. **No custody rent**
   - Revenue must not depend on holding user funds, freezing balances, or forcing users into a custodial treasury.

3. **No surveillance monetization**
   - SATNET should not fund itself through ads, tracking, behavioral profiling, or data resale.

4. **No paywall on freedom**
   - Wallet creation, backup, recovery, voucher redemption, and communication primitives must remain accessible without subscription lock-in.

5. **Forkability is a feature**
   - If the community disagrees with governance or service providers, it must be possible to fork the software and migrate.

6. **Transparent stewardship over opaque extraction**
   - Community support funds, grants, and service revenue should be documented with public reporting where operationally safe.

---

## Recommended Revenue Sources

### 1. Grants and Public-Interest Funding
Use grants for:
- protocol R&D
- security audits
- accessibility and localization
- low-end device support
- censorship-resistance research

These funds should accelerate public goods work, not create control over user funds.

### 2. Optional Federation and Partner Services
Revenue can come from optional service layers, such as:
- relay hosting
- regional directory hosting
- managed rate feeds
- compliance adapters for licensed partners
- merchant analytics dashboards
- deployment support for NGOs, cooperatives, or local operators

The key rule: the network must still function in reduced form without these paid services.

### 3. Support and Integration Contracts
SATNET can ethically earn revenue by helping others deploy it:
- implementation support
- device certification
- custom integrations
- incident-response retainers
- training for agent/merchant/verifier operators

### 4. Community Sustainers and Donations
A direct community support model is aligned with open source:
- individual donations
- recurring sustainers
- cooperative memberships
- sponsor circles for documentation, translations, and testing

### 5. Sponsored Education Without User Exploitation
Education funding is acceptable when:
- there is no surveillance advertising model
- user participation is transparent and voluntary
- sponsors do not gain control over wallet custody or protocol rules

---

## Revenue Sources to Avoid

SATNET should avoid the following unless the community explicitly chooses otherwise through transparent governance:

- mandatory protocol tolls on every transaction
- forced agent participation fees for basic access
- token issuance designed mainly for speculation
- custodial float income from pooled user balances
- advertising that profiles users
- app features that lock essential wallet freedoms behind premium plans

---

## Cost-Recovery Philosophy

The correct question is not "How much can the network extract?" but:

> "What is the minimum sustainable funding needed to keep the software secure, available, and free?"

Recommended priority order for spending:

1. security and audits
2. core maintenance
3. release engineering and QA
4. regional support and incident response
5. education and localization
6. ecosystem growth grants

---

## Governance Alignment

Instead of promising a speculative DAO future, SATNET should move through practical stewardship stages:

1. maintainer stewardship
2. public financial reporting
3. federated advisory council
4. community charter and contribution governance
5. optional federation-wide voting for non-custodial protocol governance

---

## Final Rule

If a revenue stream requires reducing user freedom, increasing custody, or adding surveillance, it is not aligned with SATNET's mission.
