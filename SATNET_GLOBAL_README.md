# SATNET GLOBAL - SATNET Mesh Bitcoin Edition
## Unified App: Decentralized Mesh Communication + Bitcoin Network for GLOBAL

**SATNET GLOBAL** is a **single integrated app** combining:
- **SATNET Mesh**: Decentralized, censorship-resistant mesh networking (voice, video, messaging)
- **Bitcoin Wallet**: Non-custodial, self-custody Bitcoin (BIP32/39/44)
- **Voucher System**: Cash-to-Bitcoin conversion through local agents
- **Lightning Network**: Fast payments to merchants
- **Reputation System**: Community-based trust & accountability

Designed for cash-based economies across **Uganda, Kenya, Tanzania, South Sudan, Ethiopia, DRC, and beyond**.

**One App. Multiple Modes. Zero Custody.**

If SATNET GLOBAL disappeared tomorrow, users would still own their Bitcoin, and the SATNET mesh network would continue functioning as a decentralized network.

# PILLAR III  
## COMMUNITY SUSTAINABILITY MODEL

SATNET GLOBAL should be sustainable in a way that is compatible with free software culture and cypherpunk values.

That means:
- no protocol tax on every user action
- no custody-based monetization
- no surveillance advertising
- no paywall on essential wallet freedoms
- no forced lock-in for agents, merchants, or users

The network should be financed like a public-good protocol with optional service layers, not like a rent-extracting platform.

---

## Sustainability Principles

### 1. Self-Custody Is Not a Premium Feature
- Wallet creation must stay free
- Backup and recovery must stay free
- Voucher redemption must not require subscription fees
- Essential communications must not be gated behind premium plans

### 2. No Mandatory Protocol Toll
- SATNET should not require a global "network cut" from every voucher or payment
- Agents may price their own local spread
- Optional partner or federation services may charge for hosted infrastructure, but not for protocol freedom itself

### 3. Revenue Comes from Services Around the Network
Sustainable revenue can come from:
- grants and public-interest funding
- implementation and support contracts
- optional hosted relay, routing, and compliance services
- merchant or partner dashboards operated as optional service layers
- community sustainers and donations
- education sponsorship that does not rely on surveillance or custody

### 4. Anti-Rent-Seeking Rule
If a revenue stream requires reducing user freedom, increasing custody, or forcing dependence on one operator, it is misaligned with SATNET's mission.

---

## Recommended Revenue Streams

### Grants and Research Funding
- Security audits
- Accessibility and localization
- Low-end device optimization
- Censorship-resistance research
- Public-interest field deployments

### Optional Federation and Partner Services
- Regional relay hosting
- Signed directory hosting
- Exchange-rate and compliance adapters
- Merchant analytics and reporting
- Training and deployment support for NGOs, cooperatives, and local operators

### Support and Integration Work
- Device certification
- Incident-response retainers
- Regional rollout support
- Partner onboarding and operational tooling

### Community Funding
- Direct donations
- Recurring sustainers
- Cooperative memberships
- Community funds for maintenance, audits, and translations

### Ethical Sponsored Education
- Learn-and-earn programs funded transparently
- No user profiling or ad-tech monetization
- No sponsor control over wallet custody or protocol rules

---

## Cost-Recovery Philosophy

SATNET should optimize for **minimum sustainable funding**, not maximum extraction.

Priority order for spending:
1. security and audits
2. core maintenance
3. release engineering and QA
4. regional support and incident response
5. education and localization
6. ecosystem growth and community grants

---

## What SATNET Should Avoid
- Mandatory protocol fees on every voucher
- Forced agent participation fees for basic access
- Token speculation as a funding model
- Custodial float income from pooled balances
- Advertising based on user surveillance
- Locking self-custody or redemption behind subscription walls

---

## Transparency Commitment
- Publish what is funded by grants, partners, or community support
- Separate protocol stewardship from partner-operated service businesses
- Preserve the user's right to fork, exit, and self-host when possible

For the detailed model, see `doc/SATNET_COMMUNITY_SUSTAINABILITY_MODEL.md`.

## Blockchain API Integration

### Esplora API
- **Purpose**: Query Bitcoin balances without full node
- **Supported Networks**: Mainnet & Testnet
- **Features**:
  - No authentication required (privacy-preserving)
  - Transaction broadcasting
  - UTXO information
  - Balance querying
- **Offline Fallback**: Local cache when API unavailable
- **Future**: Full node option for maximum privacy

### Lightning Development Kit (LDK)
- **Current Status**: Placeholder implementation (Phase 7)
- **Planned**: Full Lightning channel management (Phase 8+)
- **Features**:
  - Peer-to-peer payment channels
  - Multi-hop routing
  - Atomic swaps
  - Channel persistence
