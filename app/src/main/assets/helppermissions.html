SATNET, like all Android software, requires permissions to run. These permissions let the app interact with phone hardware, Android system services, mesh networking, and optional SATNET tools such as QR scanning and maps.
ACCESS_FINE_LOCATION / ACCESS_COARSE_LOCATION
Required on modern Android before the app can inspect nearby Wi-Fi scan results for network discovery. SATNET Maps can also use a one-time foreground device location lookup when you explicitly request it.
ACCESS_NETWORK_STATE / ACCESS_WIFI_STATE / CHANGE_WIFI_STATE / CHANGE_WIFI_MULTICAST_STATE
Used to inspect and manage Wi-Fi-related networking behavior for mesh discovery, connectivity status, and network setup.
AUTHENTICATE_ACCOUNTS / GET_ACCOUNTS / MANAGE_ACCOUNTS / READ_SYNC_SETTINGS / WRITE_SYNC_SETTINGS
Needed for the legacy Android-account integration SATNET uses for identity and contact interoperability.
BLUETOOTH / BLUETOOTH_ADMIN / BLUETOOTH_CONNECT / BLUETOOTH_SCAN / BLUETOOTH_ADVERTISE
Used for nearby Bluetooth mesh discovery, inspecting adapter state, and requesting discoverable mode on newer Android versions.
BROADCAST_STICKY
Broadcasts internal ad hoc state so SATNET UI components can react to legacy mesh status changes.
CALL_PHONE / PROCESS_OUTGOING_CALLS / READ_PHONE_STATE / READ_PHONE_NUMBERS
Used for call integration and to help prefill the user's own number or route outgoing calls through SATNET where supported.
CAMERA
Needed for QR scanning flows such as SATNET voucher redemption.
DISABLE_KEYGUARD / MODIFY_AUDIO_SETTINGS / RECORD_AUDIO / VIBRATE / WAKE_LOCK
Support calling features, notifications, and keeping the device awake while relaying or handling active communication tasks.
INTERNET
Required for general network traffic. Optional SATNET services such as exchange-rate lookups or relay-related features may also use it when enabled in the build.
POST_NOTIFICATIONS
Lets SATNET show modern Android notifications for calls, messages, and status events.
READ_CONTACTS / WRITE_CONTACTS / READ_PROFILE
Allow contact lookup, display, and insertion when integrating reachable peers with the Android contacts database.
READ_EXTERNAL_STORAGE / WRITE_EXTERNAL_STORAGE / RECEIVE_FILE / SEND_MESHMS
Support Rhizome file sharing, file receipt, and MeshMS messaging storage on devices and Android versions that still expose these storage paths.
RECEIVE_BOOT_COMPLETED
Lets SATNET restart services after device boot when appropriate.
SEND_SMS / RECEIVE_SMS / READ_SMS
Reserved for relay and fallback messaging flows where those features are enabled.
WRITE_SETTINGS
Reads or adjusts limited system settings used by notification, ringtone, and device-behavior flows where Android still allows it.
